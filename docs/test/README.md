Test PDFs used by the local password workflow live under `docs/test/generated/`.

They are generated by `node test-encryption.js` using `pdf-lib`, then validated with `pdfjs-dist` to confirm that:

- plain PDFs open normally
- password-added PDFs open with the correct password
- password-updated PDFs reject the old password and open with the new one
- password-removed PDFs open normally again
- extracted reader text still matches the fixture content through each workflow stage

Generated outputs are split into:

- `docs/test/generated/plain/`
- `docs/test/generated/encrypted/`
- `docs/test/generated/updated/`
- `docs/test/generated/unlocked/`
